# 文档评审详细规范

每个维度的检查项按优先级分为三级：
- **核心**：快速模式即启用，发现最高成本的问题
- **标准**：标准模式启用，覆盖常见遗漏
- **深度**：深度模式启用，全面覆盖

---

## 逻辑一致性 {#逻辑一致性}

### 核心检查

| 检查项 | 说明 | 后续修复成本 |
|--------|------|-------------|
| **前后描述矛盾** | 同一概念在不同位置的描述是否一致 | 高 — 实现时必须回头确认以哪个为准 |
| **流程不闭环** | 流程是否有起点但无终点或缺少中间环节 | 高 — 开发到一半发现流程走不通 |

### 深度检查

| 检查项 | 说明 | 后续修复成本 |
|--------|------|-------------|
| **数据流向矛盾** | 数据的产生、存储、使用是否逻辑连贯 | 中 — 接口联调时发现不匹配 |
| **状态定义冲突** | 同一状态在不同位置的值是否一致 | 中 — 测试阶段发现状态异常 |

### 检查方法

1. 对比相同概念在各处描述，标注矛盾
2. 绘制流程图验证闭环（起点→处理→终点）

---

## 需求清晰度 {#需求清晰度}

### 核心检查

| 检查项 | 说明 | 典型模糊词 |
|--------|------|-----------|
| **模糊词检测** | 是否包含无法量化的描述 | 适当、合理、可能、应该、尽量、一般情况下 |
| **边界不明确** | 数量、时间、范围等是否有明确限制 | "图片数量适当限制" → 应明确"最多9张" |

### 标准检查

| 检查项 | 说明 | 后续修复成本 |
|--------|------|-------------|
| **条件不完整** | 触发条件是否完整定义 | 高 — 实现时不知道什么条件下触发 |
| **角色不清晰** | 操作主体是否明确 | 中 — 开发时需反复确认用户类型 |

### 消除模糊词原则

```
❌ "图片数量适当限制"
✅ "图片最多9张，最少0张"

❌ "一般情况下用户可以编辑"
✅ "用户在帖子未被删除且为作者本人时可编辑"

❌ "适当的时间间隔"
✅ "每5秒刷新一次"
```

## 完整性 {#完整性}

### 核心检查

| 检查项 | 说明 | 后续修复成本 |
|--------|------|-------------|
| **异常分支** | 失败、错误场景是否定义 | 极高 — 生产环境直接暴露给用户 |
| **边界条件** | 极值情况是否覆盖 | 高 — 测试阶段必现 bug |

### 标准检查

| 检查项 | 说明 | 后续修复成本 |
|--------|------|-------------|
| **空状态** | 无数据时的展示是否定义 | 中 — 上线后用户看到空白页 |
| **权限边界** | 越权操作是否定义 | 高 — 安全漏洞 |

### 深度检查

| 检查项 | 说明 | 后续修复成本 |
|--------|------|-------------|
| **并发冲突** | 并发场景是否考虑 | 视场景而定 — 低频但严重 |

### 完整性检查清单

```markdown
## 功能 [功能名]
- [ ] 正向流程：[描述]
- [ ] 异常流程：
  - [ ] 网络异常：[处理方式]
  - [ ] 参数校验失败：[提示内容]
  - [ ] 服务端错误：[处理方式]
- [ ] 边界条件：
  - [ ] 最小值：[行为]
  - [ ] 最大值：[行为]
- [ ] 空状态：[展示内容]
- [ ] 权限控制：[规则]
```

## 交互闭环 {#交互闭环}

### 核心检查

| 检查项 | 说明 | 后续修复成本 |
|--------|------|-------------|
| **操作反馈** | 用户操作后是否有即时反馈 | 中 — 用户重复操作 |
| **失败反馈** | 操作失败是否有提示和恢复方案 | 高 — 用户不知道操作失败了 |
| **状态回退** | 操作取消/失败后状态是否恢复 | 高 — UI 状态与服务端不一致 |

### 标准检查

| 检查项 | 说明 | 后续修复成本 |
|--------|------|-------------|
| **加载提示** | 异步操作是否有加载状态 | 中 — 用户重复提交 |
| **成功反馈** | 操作成功是否有提示 | 低 — 体验问题 |

### 交互状态矩阵

```
| 操作 | 触发状态 | 加载状态 | 成功状态 | 失败状态 | 回退状态 |
|------|---------|---------|---------|---------|---------|
| 点赞 | 未点赞→点赞动画 | Loading图标 | 已点赞状态 | Toast提示 | 恢复未点赞 |
| 发帖 | 点击发布按钮 | 发布中动画 | Toast+跳转 | Toast+保留内容 | 内容可重试 |
```

## 移动端体验 {#移动端体验}

### 核心原则

**移动端体验优先**：所有 UI/交互设计优先考虑移动端场景。

### 核心检查

| 检查项 | 说明 | 推荐标准 |
|--------|------|---------|
| **触控目标大小** | 可点击元素是否足够大 | ≥ 44px（iOS）/ ≥ 48dp（Android） |
| **键盘弹出处理** | 输入时键盘是否遮挡关键元素 | 底栏应随键盘上移或隐藏 |
| **底部安全区** | 是否考虑底部手势区域 | 底部留出安全区 padding |

### 标准检查

| 检查项 | 说明 | 推荐标准 |
|--------|------|---------|
| **加载状态** | 是否定义加载中的展示 | 骨架屏/Loading 动画 |
| **网络状态** | 是否考虑弱网/断网场景 | 离线提示、重试机制 |

### 深度检查

| 检查项 | 说明 | 推荐标准 |
|--------|------|---------|
| **触控间距** | 相邻可点击元素间距 | ≥ 8px |
| **屏幕适配** | 不同屏幕尺寸适配 | 小屏/大屏的布局适配 |
| **手势操作** | 是否定义手势交互 | 下拉刷新、滑动删除 |
| **横竖屏** | 横屏场景处理 | 视频播放、图片查看 |

### 移动端核心清单

```markdown
## 移动端体验检查
- [ ] 触控目标 ≥ 44px
- [ ] 键盘弹出不遮挡底栏/输入框
- [ ] 底部安全区适配
- [ ] 加载状态定义（骨架屏/Loading）
- [ ] 弱网/断网处理
```

## 跨文档一致性 {#跨文档一致性}

> 仅深度模式启用，当用户提供多份文档时检查。

### 核心检查

| 检查项 | 说明 | 后续修复成本 |
|--------|------|-------------|
| **功能对齐** | 需求定义的功能在设计文档是否全覆盖 | 极高 — 开发完才发现漏了功能 |
| **字段对齐** | 接口返回字段与需求展示字段是否匹配 | 高 — 联调时才发现字段缺失 |
| **约束对齐** | 需求约束在设计实现是否遵守 | 高 — 上线后约束失效 |

### 深度检查

| 检查项 | 说明 | 后续修复成本 |
|--------|------|-------------|
| **命名一致** | 同一概念在不同文档命名是否一致 | 低 — 开发阶段自行统一 |
| **流程一致** | 需求定义的流程与设计实现是否一致 | 高 — 实现与预期不符 |

### 检查方法

1. 功能清单对比：需求功能列表 vs 设计接口列表
2. 字段映射检查：需求展示字段 vs 接口返回字段
3. 约束映射检查：需求约束 vs 接口校验逻辑

## BDD 文档生成规范 {#bdd-生成}

需求清晰度足够时，需求自然可转化为 BDD。不单独检查可测试性。

### BDD 场景格式

```markdown
## 场景：[场景名称]

**Given** [前置条件]
**When** [用户操作]
**Then** [期望结果]

### 异常场景
**Given** [前置条件]
**When** [异常操作/异常条件]
**Then** [异常处理结果]
```

### 从需求转化为 BDD

```markdown
需求描述：
用户可以在广场点击关注按钮关注发帖用户，点击已关注按钮取消关注

转化为 BDD：

## 场景：关注用户

**Given** 用户已登录，当前在广场页面，目标用户未关注
**When** 用户点击关注按钮
**Then** 关注按钮变为"已关注"状态，关注数+1

## 场景：取消关注

**Given** 用户已登录，当前在广场页面，目标用户已关注
**When** 用户点击已关注按钮
**Then** 关注按钮变为"关注"状态，关注数-1

## 场景：未登录关注（异常）

**Given** 用户未登录，当前在广场页面
**When** 用户点击关注按钮
**Then** 弹出登录引导提示
```

## 问题级别定义 {#问题级别}

### 阻塞级（Blocker）

导致实现错误或返工，必须修复：
- 逻辑矛盾导致无法实现
- 关键信息缺失无法开发
- 跨文档功能不对齐
- 异常场景未定义

### 建议级（Suggestion）

可优化但不导致返工：
- 描述可更清晰
- 边界条件可补充
- 移动端体验可优化
