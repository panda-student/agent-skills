# 说服原则理论

## 核心定义

说服原则理论（Persuasion Principles Theory）定义了如何设计更有效的技能的方法论框架。LLM对与人类相同的说服原则做出响应。

## 研究基础

Meincke等人（2025）用N=28,000个AI对话测试了7个说服原则。说服技术使合规率增加了一倍以上（33% → 72%，p < .001）。

## 七个原则

### 1. 权威（Authority）

**是什么**：对专业知识、证书或官方来源的尊重。

**在技能中如何工作：**
- 命令式语言："你必须"、"绝不"、"总是"
- 不可协商的框架："无例外"
- 消除决策疲劳和合理化

**何时使用：**
- 纪律强制技能（TDD、验证要求）
- 安全关键实践
- 既定最佳实践

**示例：**
```markdown
✅ 先写代码再写测试？删掉它。重新开始。无例外。
❌ 在可行时考虑先编写测试。
```

### 2. 承诺（Commitment）

**是什么**：与先前行动、声明或公开声明保持一致。

**在技能中如何工作：**
- 要求公告："宣布技能使用"
- 强制明确选择："选择A、B或C"
- 使用跟踪：检查清单用TodoWrite

**何时使用：**
- 确保技能实际被遵循
- 多步骤过程
- 问责机制

**示例：**
```markdown
✅ 当你找到技能时，你必须宣布："我正在使用[技能名称]"
❌ 考虑让你的伙伴知道你正在使用哪个技能。
```

### 3. 稀缺（Scarcity）

**是什么**：来自时间限制或有限可用性的紧迫性。

**在技能中如何工作：**
- 时限要求："在继续之前"
- 顺序依赖："紧接在X之后"
- 防止拖延

**何时使用：**
- 立即验证要求
- 时间敏感工作流
- 防止"我稍后做"

**示例：**
```markdown
✅ 完成任务后，立即请求代码评审再继续。
❌ 方便时可以评审代码。
```

### 4. 社会认同（Social Proof）

**是什么**：符合他人所为或被认为是正常的事物。

**在技能中如何工作：**
- 通用模式："每次"、"总是"
- 失败模式："没有Y的X = 失败"
- 建立规范

**何时使用：**
- 记录通用实践
- 警告常见失败
- 强化标准

**示例：**
```markdown
✅ 没有TodoWrite跟踪的检查清单 = 步骤被跳过。每次都是。
❌ 有些人发现TodoWrite对检查清单有帮助。
```

### 5. 统一（Unity）

**是什么**：共享身份，"我们感"，群体归属。

**在技能中如何工作：**
- 协作语言："我们的代码库"、"我们是同事"
- 共享目标："我们都想要质量"

**何时使用：**
- 协作工作流
- 建立团队文化
- 非层级实践

**示例：**
```markdown
✅ 我们是同事一起工作。我需要你诚实的技术判断。
❌ 如果我错了你可能应该告诉我。
```

### 6. 互惠（Reciprocity）

**是什么**：回报收到的利益的义务。

**如何工作：**
- 谨慎使用 - 可能有操纵性
- 技能中很少需要

**何时避免：**
- 几乎总是（其他原则更有效）

### 7. 喜好（Liking）

**是什么**：偏好与我们喜欢的人合作。

**如何工作：**
- **不要用于合规**
- 与诚实反馈文化冲突
- 创建阿谀奉承

**何时避免：**
- 总是用于纪律强制

## 按技能类型的组合

| 技能类型 | 使用 | 避免 |
|---------|------|------|
| 纪律强制 | 权威 + 承诺 + 社会认同 | 喜好、互惠 |
| 指导/技术 | 适度权威 + 统一 | 重度权威 |
| 协作 | 统一 + 承诺 | 权威、喜好 |
| 参考 | 仅清晰 | 所有说服 |

## 为什么有效：心理学

**明线规则减少合理化：**
- "你必须"消除决策疲劳
- 绝对语言消除"这是例外吗？"问题
- 明确的反合理化堵住特定漏洞

**实施意图创建自动行为：**
- 清晰触发器 + 要求行动 = 自动执行
- "当X，做Y"比"通常做Y"更有效
- 减少合规的认知负荷

**LLM是准人类的：**
- 在包含这些模式的人类文本上训练
- 权威语言在训练数据中先于合规
- 承诺序列（声明 → 行动）经常建模
- 社会认同模式（每个人都做X）建立规范

## 道德使用

**合法：**
- 确保关键实践被遵循
- 创建有效文档
- 防止可预测的失败

**不合法：**
- 为个人利益操纵
- 制造虚假紧迫
- 基于内疚的合规

**测试：** 如果用户完全理解，这项技术是否符合用户的真正利益？

## 关键要点

1. **权威** - 使用命令式语言
2. **承诺** - 要求公告和明确选择
3. **稀缺** - 创建紧迫性
4. **社会认同** - 建立规范
5. **统一** - 使用协作语言
6. **道德使用** - 服务用户的真正利益
