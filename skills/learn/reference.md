# learn 技能参考

## 核心目标

**帮助后续更高效开发，提升开发效率**

通过固化可复用知识，避免重复踩坑，加速后续开发。

---

## PDCA 循环

复盘流程遵循 PDCA 循环，确保持续优化：

```
┌─────────────────────────────────────────────────────────────┐
│  P (Plan)    →  会话回顾，确定复盘目标                        │
│  D (Do)      →  经验提取，分类整理                            │
│  C (Check)   →  展示建议，用户确认                            │
│  A (Act)     →  执行更新，固化标准化                          │
└─────────────────────────────────────────────────────────────┘
          ↓
    下一轮 PDCA（持续优化）
```

### PDCA 各阶段要点

| 阶段 | 核心动作 | 输出 |
|------|---------|------|
| Plan | 会话回顾、流程复盘、识别关键点 | 复盘目标清单 |
| Do | 经验提取、分类判断 | 分类后的经验列表 |
| Check | 展示建议、获取确认 | 用户确认的固化清单 |
| Act | 执行固化、更新文档 | 更新后的技能/记忆 |

## 流程优化固化

### 识别条件
当会话中出现以下情况时，需提取流程优化经验：
- 完成一个完整任务后，回顾执行效率
- 发现某些步骤组合效果更好
- 发现某些步骤导致延误或错误
- 总结出下次同类任务的最佳执行顺序

### 固化目标
流程优化经验应固化到**对应任务的技能**中：

| 任务类型 | 固化目标技能 | 固化位置 |
|---------|-------------|---------|
| JPM开发任务 | jpm-task-flow | SKILL.md 流程部分 |
| 调试排查任务 | systematic-debugging | SKILL.md 流程部分 |
| 代码评审任务 | code-review | reference.md |
| 测试任务 | test-driven-development | reference.md |
| 其他任务类型 | 对应技能或新建技能 | 视情况 |

### 固化内容
提取并记录：
1. **任务类型**：什么类型的任务
2. **推荐顺序**：最佳执行步骤顺序
3. **优化点**：可省略、可并行、可提前准备的步骤
4. **避坑提醒**：容易出错的步骤及应对方法

### 判断示例
```
✅ JPM开发任务执行流程：
   原流程：直接改代码 → 测试失败 → 查API → 重改 → 测试通过
   优化后：先查API文档 → 理解业务规则 → 编写代码 → 测试验证
   
   → 固化到 jpm-task-flow：
     更新最优执行流程模板

✅ 调试排查任务执行流程：
   原流程：猜测问题 → 改代码 → 测试失败 → 再猜测...
   优化后：收集信息 → 分析日志 → 定位根因 → 制定方案 → 验证修复
   
   → 固化到 systematic-debugging：
     添加最优排查流程
```

## 新发现固化

### 识别条件
当会话中出现以下情况时，需提取并固化：
- **新接口**：发现新的 API 端点、服务接口、系统能力
- **新能力/方法**：发现新的工具用法、库函数、快捷操作
- **新配置/参数**：发现新的配置项、环境变量、参数组合

### 固化目标
新发现应固化到**对应的技能**中：

| 新发现类型 | 固化目标技能 | 固化位置 |
|-----------|-------------|---------|
| JPM系统新接口 | jpm-api-reference | reference.md |
| 系统新能力/机制 | jpm-architecture | reference.md |
| 新业务规则 | jpm-business | reference.md |
| 通用工具/方法 | 对应工具技能 | examples.md |
| 其他项目特定 | 新建技能或 Memory | 视情况 |

### 固化内容
提取并记录：
1. **发现来源**：在什么场景下发现的
2. **功能用途**：能解决什么问题、提升什么效率
3. **使用方式**：具体调用方法、参数配置
4. **注意事项**：限制条件、依赖关系

### 判断示例
```
✅ 发现：/api/v2/batch-export 接口可批量导出数据
   → 固化到 jpm-api-reference：
     添加接口文档，说明用途和调用方式

✅ 发现：SIGN插件可通过配置实现动态校验
   → 固化到 jpm-architecture：
     添加 SIGN 插件进阶用法说明

✅ 发现：PowerShell Get-Content -Tail 参数可实时读取日志
   → 固化到新建 "log-monitoring" 技能或现有相关技能
```

## 命令执行经验固化

### 识别条件
当会话中出现以下情况时，需提取命令执行经验：
- 同一命令执行 2 次以上才成功
- 命令执行报错后调整参数才成功
- 特定环境/前置条件下命令才能成功

### 固化目标
命令经验应固化到**执行该命令的技能**中，而非创建新技能：

| 命令类型 | 固化目标技能 | 固化位置 |
|---------|-------------|---------|
| curl 请求 | jpm-api-reference, zentao-operate 等 | examples.md |
| git 操作 | 相关 git 技能 | reference.md |
| 文件操作 | 相关文件处理技能 | reference.md |
| 其他 | 对应调用该命令的技能 | examples.md |

### 固化内容
提取并记录：
1. **原始命令**：最初失败的命令
2. **成功命令**：最终成功的完整命令
3. **关键差异**：参数、环境变量、前置条件的差异
4. **错误信息**：失败时的报错信息
5. **解决方案**：为什么成功的命令有效

### 判断示例
```
❌ 第一次：curl -X GET "http://api/data"  # 报错：连接超时
❌ 第二次：curl -X GET "http://api/data" --connect-timeout 30  # 报错：SSL证书问题
✅ 第三次：curl -X GET "http://api/data" --connect-timeout 30 -k  # 成功

→ 固化到 jpm-api-reference 技能：
  添加 --connect-timeout 30 -k 参数说明
```

## 经验分类判断

### 经验去向优先级（按固化价值排序）

1. **流程优化固化**（最高优先）：可复用的最优执行流程 → 更新到对应技能
2. **新发现固化**：新接口、新能力、新方法 → 更新到对应技能
3. **命令经验固化**：多次尝试才成功的命令 → 更新到对应技能
4. **新建技能**：跨项目可用的独立流程/方法 → 创建新技能
5. **更新技能**：对现有技能的补充/修正 → 更新现有技能
6. **记录Memory**：项目内跨会话可用的上下文/偏好 → 写入 memory

### 判断问题

- 这是最优执行流程吗？→ 流程优化固化
- 这是新发现的接口或能力吗？→ 新发现固化
- 这是命令执行多次尝试的经验吗？→ 命令经验固化
- 这个经验在其他项目中也能用吗？→ 新建技能或更新技能
- 这个经验只与当前项目相关吗？→ Memory
- 这是关于项目状态或用户偏好的事实吗？→ Memory

### 技能 vs Memory

| 类型 | 适用范围 | 典型内容 |
|------|---------|---------|
| 技能 | 跨项目可用 | 流程、方法、判断逻辑、最佳实践 |
| Memory | 项目内跨会话可用 | 项目上下文、用户偏好、临时状态、一次性事实 |

### 新技能判断

创建新技能需满足：
- **复用性**：未来可能多次使用
- **独立性**：与现有技能有明确区分
- **完整性**：能形成独立的知识单元

不满足时考虑：
- 更新现有技能的 reference.md 或 examples.md
- 记录为 Memory

## 整理现有知识

### 整理触发条件

- 手动触发：用户说"整理一下技能"或"整理一下记忆"
- 复盘时顺带检查是否有需要整理的内容

### 整理动作

**技能整理：**
- **合并**：发现重复或高度相似的技能
- **拆分**：发现职责过多的大型技能
- **清理**：发现过时或不再使用的技能
- **补充**：发现 reference.md 或 examples.md 缺失

**Memory 整理：**
- **合并**：发现重复或相关的记忆条目
- **清理**：发现过时或不再相关的记忆
- **更新**：发现记忆内容已过时需要修正

### 整理确认

整理前必须展示建议并获取用户确认：
```
发现以下可整理的内容：

【技能整理】
- xxx 和 yyy 功能相似，建议合并
- zzz 已过时，建议清理

【Memory 整理】
- aaa 和 bbb 相关，建议合并
- ccc 已过时，建议清理

是否执行整理？(Y/n/逐项确认)
```
