# learn 技能参考

## 经验分类判断

### 技能 vs Memory

| 类型 | 适用范围 | 典型内容 |
|------|---------|---------|
| 技能 | 跨项目可用 | 流程、方法、判断逻辑、最佳实践 |
| Memory | 项目内跨会话可用 | 项目上下文、用户偏好、临时状态、一次性事实 |

### 判断问题

- 这个经验在其他项目中也能用吗？→ 技能
- 这个经验只与当前项目相关吗？→ Memory
- 这是一条可复用的流程/方法吗？→ 技能
- 这是关于项目状态或用户偏好的事实吗？→ Memory

### 新技能判断

创建新技能需满足：
- **复用性**：未来可能多次使用
- **独立性**：与现有技能有明确区分
- **完整性**：能形成独立的知识单元

不满足时考虑：
- 更新现有技能的 reference.md 或 examples.md
- 记录为 Memory

## 整理现有知识

### 整理触发条件

- 手动触发：用户说"整理一下技能"或"整理一下记忆"
- 复盘时顺带检查是否有需要整理的内容

### 整理动作

**技能整理：**
- **合并**：发现重复或高度相似的技能
- **拆分**：发现职责过多的大型技能
- **清理**：发现过时或不再使用的技能
- **补充**：发现 reference.md 或 examples.md 缺失

**Memory 整理：**
- **合并**：发现重复或相关的记忆条目
- **清理**：发现过时或不再相关的记忆
- **更新**：发现记忆内容已过时需要修正

### 整理确认

整理前必须展示建议并获取用户确认：
```
发现以下可整理的内容：

【技能整理】
- xxx 和 yyy 功能相似，建议合并
- zzz 已过时，建议清理

【Memory 整理】
- aaa 和 bbb 相关，建议合并
- ccc 已过时，建议清理

是否执行整理？(Y/n/逐项确认)
```
