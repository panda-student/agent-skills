## 示例 1：需求文档评审

### 输入文档片段

```markdown
## 发帖功能

用户可以在广场页面点击发布按钮，填写帖子标题和正文，上传图片，然后发布帖子。
图片数量适当限制，帖子内容应该合理。

页面元素：
- 发布按钮：广场页面右下角悬浮图标
- 输入框：标题输入框、正文输入框
- 图片上传按钮

用户操作：
点击发布 → 填写内容 → 点击发布按钮 → 帖子发布成功
```

### 评审结果

#### 修改记录（已自动修复）

| 位置 | 原内容 | 修改后 | 原因 | 级别 |
|------|--------|--------|------|------|
| 图片数量限制 | "适当限制" | "最多9张，最少0张" | 消除模糊词 | 阻塞 |
| 内容限制 | "应该合理" | "标题1-100字，正文1-20000字" | 消除模糊词 | 阻塞 |
| 发布按钮位置 | "右下角悬浮" | "右下角悬浮图标（write.svg），距底部安全区16px" | 补充移动端适配 | 建议 |

#### 待决策问题

1. **发帖成功后的行为**
   - A: Toast提示"发布成功" + 返回广场刷新列表
   - B: Toast提示 + 跳转到帖子详情页
   - C: Toast提示 + 保留在发帖页（可继续发帖）
   
   **建议**：A，符合用户预期，可立即看到发布的帖子

2. **发布失败后的处理**
   - A: Toast提示失败原因 + 保留填写内容 + 提供重试按钮
   - B: Toast提示 + 清空内容 + 返回广场
   
   **建议**：A，避免用户重复填写，体验更友好

#### 完整性遗漏

- [ ] 未定义：空标题/空正文提交时的校验提示
- [ ] 未定义：图片上传失败的处理
- [ ] 未定义：发布过程中的 Loading 状态
- [ ] 未定义：网络异常时的处理

#### BDD 可转化性

- ✅ 可转化：发布成功流程
- ⚠️ 需澄清：图片上传失败流程、网络异常流程

## 示例 2：移动端体验评审

### 输入文档片段

```markdown
## 帖子详情页底部栏

页面底部固定一个操作栏，左侧是评论输入框，右侧是点赞、收藏、评论、转发图标。
用户点击输入框可以弹出键盘进行评论输入。
```

### 评审结果

#### 移动端体验问题

| 问题 | 描述 | 建议 | 级别 |
|------|------|------|------|
| 键盘遮挡 | 未定义键盘弹出时底部栏的处理 | 底部栏随键盘上移，或改为固定在键盘上方 | 阻塞 |
| 安全区 | 未考虑底部手势区域 | 底部栏距屏幕底部预留 34px（iOS）或 20dp（Android）安全区 | 阻塞 |
| 触控区域 | 未定义图标大小 | 点赞/收藏等图标触控区域 ≥ 44px | 建议 |
| 输入框高度 | 输入框高度未定义 | 输入框高度 36-44px，触控友好 | 建议 |

#### 补充内容（已自动修复）

```markdown
## 帖子详情页底部栏（已修复）

页面底部固定一个操作栏：
- 左侧：评论输入框（高度40px，placeholder："发布评论"）
- 右侧：点赞、收藏、评论、转发图标（每个触控区域≥44px，间距8px）
- 底部安全区：预留34px（iOS）/ 20dp（Android）

键盘弹出处理：
- 操作栏随键盘上移，保持在键盘上方可见
- 或：操作栏隐藏，键盘上方显示简化输入栏

交互状态：
- 点击输入框 → 弹出键盘 + 输入框获得焦点 + 显示发送按钮
- 输入过程中 → 显示字数统计（上限200字）
- 点击发送 → Loading状态 + 发送成功后Toast提示 + 清空输入框 + 键盘收起
```

## 示例 3：跨文档一致性评审

### 输入：需求文档 + 设计文档

#### 需求文档定义

```markdown
## 关注功能

- 用户可在广场帖子卡片点击"关注"按钮关注作者
- 用户可在帖子详情页点击"关注"按钮关注作者
- 点击后按钮变为"已关注"状态
- 关注数实时更新
```

#### 设计文档定义

```markdown
## 关注接口

POST /api/forum/follow/add
Body: { targetUserId: number }
Response: { ok: boolean }

POST /api/forum/follow/remove
Body: { targetUserId: number }
Response: { ok: boolean }
```

### 跨文档评审结果

| 检查项 | 需求定义 | 设计定义 | 问题 | 级别 |
|--------|---------|---------|------|------|
| 功能覆盖 | 广场+详情页关注 | 仅定义关注接口 | ✅ 一致 | - |
| 状态返回 | 按钮变为"已关注" | Response仅返回ok | 缺少关注状态返回 | 阻塞 |
| 数量更新 | 关注数实时更新 | 接口无关注数返回 | 缺少关注数返回 | 阻塞 |
| 异常处理 | 未定义 | 未定义 | 需补充 | 建议 |

#### 建议接口修改

```markdown
## 关注接口（建议修改）

POST /api/forum/follow/add
Body: { targetUserId: number }
Response: 
  - ok: boolean
  - followed: boolean（当前关注状态）
  - followerCount: number（目标用户关注数）

## 关注接口异常场景补充

- 未登录：返回 FORUM_40101，前端弹出登录引导
- 重复关注：返回 FORUM_40901，前端按钮状态不变
- 关注自己：返回 FORUM_40001，前端提示"不能关注自己"
```

## 示例 4：BDD 文档生成

### 输入需求文档

```markdown
## 评论功能

用户可以在帖子详情页底部输入框输入评论内容，点击发送按钮提交评论。
评论成功后，评论列表实时刷新，评论数+1。
评论内容限制1-500字，可包含图片最多3张。
```

### 生成的 BDD 文档

```markdown
# 论坛模块 BDD 场景 - 评论功能

## 场景：发表评论（正向）

**Given** 用户已登录，当前在帖子详情页，帖子状态正常
**When** 用户在底部输入框输入评论内容（1-500字），点击发送按钮
**Then** 
- 显示发送Loading状态
- 评论提交成功
- Toast提示"评论成功"
- 评论列表刷新，新增评论出现在列表中
- 帖子评论数+1
- 输入框清空

## 场景：发表带图片评论

**Given** 用户已登录，当前在帖子详情页
**When** 用户输入评论内容，上传1-3张图片，点击发送
**Then** 
- 评论提交成功
- 评论内容显示文本+图片

## 场景：评论字数超限（校验失败）

**Given** 用户已登录，当前在帖子详情页
**When** 用户输入评论内容超过500字，点击发送
**Then** 
- 发送按钮禁用或点击时Toast提示"评论内容不能超过500字"
- 评论不提交
- 输入框保留内容，用户可修改

## 场景：评论内容为空（校验失败）

**Given** 用户已登录，当前在帖子详情页
**When** 用户输入框为空（0字），点击发送
**Then** 
- Toast提示"请输入评论内容"
- 评论不提交

## 场景：未登录评论（权限异常）

**Given** 用户未登录，当前在帖子详情页
**When** 用户点击评论输入框或发送按钮
**Then** 
- 弹出登录引导弹窗或跳转登录页

## 场景：评论发送失败（网络异常）

**Given** 用户已登录，当前在帖子详情页，网络不稳定
**When** 用户点击发送按钮，请求超时或失败
**Then** 
- Toast提示"发送失败，请重试"
- 输入框保留内容
- 提供重试按钮或自动重试机制

## 场景：帖子已删除（状态异常）

**Given** 用户已登录，帖子已被作者删除
**When** 用户尝试发送评论
**Then** 
- Toast提示"帖子不存在"
- 禁止评论提交

## 场景：图片上传失败

**Given** 用户已登录，尝试上传评论图片
**When** 图片上传失败（网络/格式/大小问题）
**Then** 
- 图片上传位置显示失败状态
- Toast提示失败原因
- 提供重试上传或删除图片选项
```
